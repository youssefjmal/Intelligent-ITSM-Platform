# Definition of Done

An item is done only if all checks below are true.

## Quality checklist

- Code is merged with clean, understandable structure.
- Existing behavior is preserved unless change is explicitly requested.
- Relevant tests pass locally.
- No sensitive secrets are committed.
- API contracts are unchanged or documented.
- Logging/error messages are actionable.
- Edge cases and fallbacks are handled.
- Documentation is updated (if behavior/process changed).

## Release readiness

- Branch is pushed to remote.
- Team can reproduce and verify the feature.
- Open risks are listed in the PR or handoff note.
